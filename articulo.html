Modelo OSI

Capa Física (Physical Layer)

La Capa Física del modelo OSI constituye el nivel más básico de la arquitectura, encargándose de la transmisión física de bits a través de medios de comunicación como cables de cobre, fibra óptica o señales de radio. Sus responsabilidades incluyen la definición de las características eléctricas, mecánicas y funcionales de los dispositivos de red, así como la especificación de la topología física. Además, se ocupa de la sincronización de la transmisión de bits entre los dispositivos conectados y la modulación de señales para facilitar la comunicación efectiva. En esencia, la Capa Física establece las bases para el transporte de datos, garantizando una conexión física robusta y confiable entre los nodos de la red.

Protocolos

Ethernet: Utilizado comúnmente en redes locales (LAN), Ethernet define los estándares para la transmisión de datos a través de cables de cobre.

USB (Universal Serial Bus): Aunque USB es conocido principalmente como un estándar para la conexión de dispositivos periféricos, también incluye especificaciones físicas para la conexión de cables y conectores.

Fibra óptica: No es un protocolo específico, pero la fibra óptica se utiliza en la Capa Física para transmitir datos a través de pulsos de luz. Los estándares asociados con la fibra óptica, como el estándar de Ethernet para fibra, están relacionados con la Capa Física.


Capa de Enlace de Datos (Data Link Layer)

La Capa de Enlace de Datos, ubicada en el segundo nivel del modelo OSI, desempeña un papel crucial en la transmisión de datos entre nodos directamente conectados en una red. Su función principal es garantizar una comunicación fiable y libre de errores en el enlace físico, dividiéndose en dos subcapas: Control de Acceso al Medio (MAC) y Control de Enlace Lógico (LLC). La subcapa MAC se ocupa de la asignación única de direcciones físicas (MAC addresses) a dispositivos de red para facilitar la identificación, mientras que la subcapa LLC se enfoca en el control de flujo y la corrección de errores. En conjunto, la Capa de Enlace de Datos supervisa la correcta transmisión de tramas de datos, coordina el acceso al medio compartido y proporciona la base esencial para la conectividad eficiente entre nodos adyacentes.

Protocolos

PPP(Point-to-Point Protocol): Este protocolo se utiliza comúnmente para establecer conexiones punto a punto, como las que se encuentran en conexiones de módem discado y conexiones de línea dedicada. PPP gestiona la encapsulación de datos y proporciona la autenticación y el control de enlace.

HDLC (High-Level Data Link Control): HDLC es un protocolo de enlace de datos que se utiliza para la transmisión de datos sobre líneas seriales. Es un protocolo síncrono que define cómo los datos se encapsulan y transmiten en tramas. HDLC ha sido fundamental en el desarrollo de otros protocolos, como el PPP mencionado anteriormente.

Ethernet: Aunque también se mencionó anteriormente en relación con la Capa Física, Ethernet opera en la Capa de Enlace de Datos para gestionar la comunicación entre dispositivos en una red local (LAN). Define cómo los datos se encapsulan y transmiten en tramas a través de medios compartidos.


Capa de Red (Network Layer)

La Capa de Red, ubicada en el tercer nivel del modelo OSI, desempeña un papel esencial en el enrutamiento de datos a través de una red. Su función principal consiste en determinar la ruta más eficiente para enviar paquetes de datos desde el origen hasta el destino, teniendo en cuenta la topología de la red y las direcciones lógicas. Además, la Capa de Red es responsable de fragmentar y reensamblar los datos, así como de asignar direcciones lógicas únicas a los dispositivos mediante protocolos como el Protocolo de Internet (IP). Al utilizar enrutadores, esta capa facilita la comunicación entre diferentes redes y constituye un elemento fundamental para la interoperabilidad en el ámbito de las redes de computadoras.

Protocolos

OSPF (Open Shortest Path First): OSPF es un protocolo de enrutamiento de estado de enlace utilizado en redes IP. Su función es determinar las rutas más eficientes entre routers, teniendo en cuenta factores como la velocidad del enlace y la congestión de la red. OSPF es especialmente común en redes empresariales y de proveedores de servicios.

ICMP (Internet Control Message Protocol): Aunque técnicamente es un subprotocolo de IP, ICMP opera en la Capa de Red y se utiliza para enviar mensajes de control y error entre dispositivos de red. Es esencial para diagnóstico de red, ya que incluye comandos como "ping" para verificar la conectividad.

IP (Internet Protocol): Este es quizás el protocolo más fundamental de la Capa de Red. IP se encarga de asignar direcciones lógicas únicas a los dispositivos en una red y facilita el enrutamiento de paquetes de datos entre redes distintas. IPv4 e IPv6 son dos versiones principales de este protocolo.


Capa de Transporte (Transport Layer)

La Capa de Transporte, situada en el cuarto nivel del modelo OSI, es esencial para el transporte eficiente y confiable de datos entre aplicaciones en sistemas finales. Su función principal es proporcionar servicios de comunicación extremo a extremo, asegurando una entrega de datos sin errores, la segmentación y reensamblaje de mensajes, y la gestión del control de flujo. Esta capa utiliza dos protocolos principales, TCP (Protocolo de Control de Transmisión) para conexiones confiables y orientadas a la conexión, y UDP (Protocolo de Datagramas de Usuario) para conexiones no confiables y sin conexión, como las utilizadas en transmisiones de audio y video. Además, la Capa de Transporte juega un papel crucial en la corrección de errores, garantizando que los datos se entreguen correctamente y en el orden adecuado, proporcionando así una base sólida para las aplicaciones que operan en las capas superiores del modelo OSI.

Protocolos

UDP (Protocolo de Datagramas de Usuario): A diferencia de TCP, UDP es un protocolo sin conexión y no confiable. Se utiliza cuando la velocidad de transmisión y la eficiencia son prioritarias sobre la garantía de entrega. Aplicaciones como transmisiones en tiempo real, videoconferencias y juegos en línea a menudo utilizan UDP.

SCTP (Protocolo de Control de Transmisión Simplificado): SCTP es un protocolo de transporte que combina características de TCP y UDP. Proporciona la entrega ordenada de datos y la corrección de errores, como TCP, pero también permite la transmisión de datos en paralelo, similar a UDP. SCTP es utilizado en algunas aplicaciones que requieren tanto la fiabilidad como la eficiencia de la transmisión simultánea de datos, como servicios de telefonía sobre IP y transmisión de video en tiempo real.

TCP (Protocolo de Control de Transmisión): TCP es un protocolo orientado a la conexión y confiable que proporciona una entrega de datos garantizada, la corrección de errores, y la reordenación de paquetes en caso de que lleguen fuera de secuencia. Es ampliamente utilizado para aplicaciones que requieren una transmisión de datos precisa y sin pérdidas, como la transferencia de archivos y la navegación web.


Capa de Sesión (Session Layer)

La Capa de Sesión, situada en el quinto nivel del modelo OSI, despliega un papel crucial en el establecimiento, gestión y finalización de sesiones entre aplicaciones en sistemas finales. Su función principal es facilitar la comunicación entre procesos de software en diferentes dispositivos, garantizando la sincronización y el control del diálogo entre ellos. La Capa de Sesión se ocupa de aspectos como la establecimiento, mantenimiento y cierre de conexiones, así como la gestión de la sincronización y la recuperación en caso de fallos. Además, proporciona servicios de diálogo, permitiendo que las aplicaciones se comuniquen de manera bidireccional o en grupos. Aunque algunas de sus funciones pueden ser asumidas por las capas superiores en ciertos modelos de red, la Capa de Sesión juega un papel fundamental en la creación y supervisión de sesiones que facilitan la transferencia de datos coherente y segura entre aplicaciones.

Protocolos (Esta capa no tiene protocolos en especifico como otras)

PPTP (Protocolo de túnel de punto a punto): PPTP opera principalmente en la Capa de Enlace de Datos, pero también incluye funcionalidades de la Capa de Sesión. Se utiliza para establecer conexiones VPN (Redes Privadas Virtuales) y facilita la creación y el mantenimiento de sesiones seguras entre redes remotas.

RPC (Remote Procedure Call): Aunque RPC abarca varias capas del modelo OSI, incluyendo la Capa de Sesión, facilita la comunicación entre procesos en diferentes sistemas mediante llamadas a procedimientos remotos. RPC contribuye al establecimiento y gestión de sesiones entre aplicaciones distribuidas.

NetBIOS (Sistema básico de entrada/salida de red): Aunque NetBIOS opera en capas superiores, proporciona servicios de sesión que facilitan el establecimiento y el cierre de sesiones entre dispositivos en una red. Es comúnmente utilizado en redes Windows.


Capa de Presentación (Presentation Layer)

La Capa de Presentación, ubicada en el sexto nivel del modelo OSI, desempeña un papel esencial en la interoperabilidad de sistemas finales al garantizar que los datos sean presentados y entendidos de manera coherente entre aplicaciones. Esta capa se encarga de la traducción y conversión de formatos de datos, la compresión y descompresión, así como el cifrado y descifrado necesario para la seguridad de la comunicación. Al estandarizar la representación de datos, la Capa de Presentación facilita la comunicación entre sistemas con diferentes codificaciones y formatos. Además, realiza funciones de gestión de sesiones, como la sincronización y el control de diálogo, contribuyendo así a una transmisión eficiente y fiable de información entre aplicaciones en sistemas distribuidos.

Protocolos

ASCII (American Standard Code for Information Interchange): Aunque más un conjunto de caracteres que un protocolo, ASCII es esencial en la representación de texto y caracteres en la Capa de Presentación. Define códigos numéricos para caracteres alfabéticos, numéricos y especiales utilizados en la comunicación de texto.

JPEG (Joint Photographic Experts Group): JPEG es un estándar de compresión de imágenes que afecta la forma en que las imágenes se presentan y transmiten. Aunque su implementación está más centrada en la compresión de datos, tiene implicaciones en cómo se representan y visualizan las imágenes.

SSL/TLS (Secure Sockets Layer/Transport Layer Security): Aunque a menudo se asocia con la Capa de Seguridad (que a veces se considera una subcapa de la Capa de Presentación), SSL y su sucesor TLS proporcionan cifrado y autenticación para la seguridad de la comunicación en capas superiores, incluyendo la presentación de datos de manera segura.


Capa de Aplicación (Application Layer)

La Capa de Aplicación, la última capa del modelo OSI, actúa como interfaz entre el usuario y la red, proporcionando servicios y funciones que son directamente accesibles a las aplicaciones y servicios de software. Su objetivo principal es permitir a las aplicaciones comunicarse a través de la red, brindando servicios específicos para necesidades como correo electrónico, transferencia de archivos, acceso a bases de datos y navegación web. Aunque no es una única entidad monolítica, abarca una amplia variedad de protocolos y servicios específicos de aplicación, como HTTP para la World Wide Web, SMTP para el correo electrónico, y FTP para la transferencia de archivos. La Capa de Aplicación juega un papel crucial en la creación de un entorno de red que satisface las diversas demandas de las aplicaciones y usuarios finales, facilitando la comunicación y el intercambio de información a través de sistemas distribuidos.

Protocolos

HTTP (Hypertext Transfer Protocol): Utilizado para la transferencia de información en la World Wide Web, HTTP permite la comunicación entre clientes (navegadores web) y servidores. Es fundamental para la visualización de páginas web y la transmisión de datos en la internet.

SMTP (Simple Mail Transfer Protocol): Diseñado para el envío de correos electrónicos, SMTP facilita la transferencia de mensajes de correo electrónico entre servidores de correo. Es esencial para la comunicación por correo electrónico y se encarga de enviar mensajes a través de la red.

FTP (File Transfer Protocol): FTP es utilizado para la transferencia de archivos entre sistemas en una red. Permite a los usuarios cargar y descargar archivos desde y hacia servidores remotos. Es comúnmente utilizado para administrar y compartir archivos en entornos de red.
